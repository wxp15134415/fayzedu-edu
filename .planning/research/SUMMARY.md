### 2. 技术栈验证

**结论**: 当前技术栈无需重大变更

- Web 框架: Axum 0.8 ✅
- ORM: SQLx 0.8 ✅
- 认证: jsonwebtoken 9 + argon2 0.5 ✅
- 验证: validator 0.18 ✅
- 日志: tracing ✅

**可选改进**:
- 添加 `axum-valid` 简化请求验证
- 添加 `utoipa` 生成 API 文档

### 3. 架构模式

**组件边界**:
```
Router (main.rs) → API Handlers → Models → Database
```

**数据流**: HTTP 请求 → Router 匹配 → State 注入 → Handler 处理 → Error 转换 → HTTP 响应

**构建顺序**:
- Phase 1: 基础设施层（error、db、middleware）
- Phase 2: 基础业务层（users、roles、dicts）
- Phase 3-6: 核心业务层（按依赖关系递增）

### 4. 关键陷阱预警

| 陷阱 | 影响 | 预防措施 |
|------|------|---------|
| 成绩批量录入事务不完整 | 数据不一致 | 使用 SQLx 事务 |
| 权限控制粒度过粗 | 越权访问 | 实现行级权限 |
| 数据关系一致性未验证 | 孤立数据 | 删除前检查关联 |
| 敏感数据未脱敏 | 隐私泄露 | 身份证/电话脱敏 |
| API 响应格式不一致 | 客户端难处理 | 统一 Error 类型 |
