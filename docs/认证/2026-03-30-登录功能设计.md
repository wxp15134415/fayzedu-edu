# Fayzedu 登录功能设计方案

> 创建日期：2026-03-30

## 1. 技术方案确认

| 项目 | 技术选型 |
|------|----------|
| 架构 | Multi-repo（多仓库） |
| 前端 | Vue3 + Naive UI + Nova Admin |
| 后端 | Rust + Axum |
| 桌面端 | Tauri（复用前端代码） |
| 数据库 | SQLite（本地）+ PostgreSQL（云端） |
| 离线存储 | SQLite + Tauri Store |
| 同步 | 模块化 Sync Adapter |
| 认证 | JWT |
| API | RESTful |

## 2. 认证流程设计

### 2.1 在线登录

```
用户输入用户名密码
        │
        ▼
前端 POST /api/auth/login
        │
        ▼
后端验证密码（argon2）
        │
        ▼
返回 JWT tokens
        │
        ▼
前端存储 tokens，跳转主页
```

### 2.2 Token 设计

| Token | 有效期 | 用途 |
|-------|--------|------|
| access_token | 15分钟 | API 请求认证 |
| refresh_token | 7天 | 刷新 access_token |
| offline_token | 30天 | 桌面端离线登录 |

### 2.3 离线登录（桌面端）

```
首次登录成功
        │
        ▼
后端颁发 offline_token（30天）
        │
        ▼
存储到 Tauri Store
        │
        ▼
离线期间：使用 offline_token 本地验证
        │
        ▼
联网后：自动调用 /api/auth/refresh 同步状态
```

## 3. 数据库设计

### 3.1 后端数据库（PostgreSQL/SQLite）

```sql
CREATE TABLE users (
    id TEXT PRIMARY KEY,
    username TEXT UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);

CREATE TABLE token_blacklist (
    token_id TEXT PRIMARY KEY,
    revoked_at INTEGER NOT NULL
);
```

### 3.2 本地数据库（桌面端 SQLite）

```sql
CREATE TABLE local_users (
    id TEXT PRIMARY KEY,
    username TEXT NOT NULL,
    offline_token_hash TEXT NOT NULL,
    last_sync_at INTEGER
);
```

## 4. API 设计

### 4.1 登录接口

```
POST /api/auth/login
Content-Type: application/json

Request:
{
    "username": "admin",
    "password": "password123"
}

Response (200):
{
    "code": 0,
    "message": "登录成功",
    "data": {
        "access_token": "eyJ...",
        "refresh_token": "eyJ...",
        "offline_token": "eyJ...",
        "expires_in": 900,
        "user": {
            "id": "uuid",
            "username": "admin"
        }
    }
}
```

### 4.2 Token 刷新

```
POST /api/auth/refresh
Content-Type: application/json

Request:
{
    "refresh_token": "eyJ..."
}

Response (200):
{
    "code": 0,
    "message": "刷新成功",
    "data": {
        "access_token": "eyJ...",
        "refresh_token": "eyJ...",
        "expires_in": 900
    }
}
```

### 4.3 登出

```
POST /api/auth/logout
Authorization: Bearer <access_token>

Response (200):
{
    "code": 0,
    "message": "登出成功"
}
```

### 4.4 获取当前用户

```
GET /api/auth/me
Authorization: Bearer <access_token>

Response (200):
{
    "code": 0,
    "message": "成功",
    "data": {
        "id": "uuid",
        "username": "admin"
    }
}
```

## 6. 实现步骤

### Phase 1: 后端基础
1. 初始化 Rust 项目
2. 配置 Axum 框架
3. 实现用户模型和数据库
4. 实现登录 API
5. 实现 JWT 颁发和验证
6. 实现 Token 刷新 API

### Phase 2: 前端基础
1. 配置 API 服务
2. 实现登录页面
3. 实现 Token 管理（存储、自动刷新）
4. 实现路由守卫

### Phase 3: 桌面端
1. 初始化 Tauri 项目
2. 配置离线存储
3. 实现离线登录逻辑
4. 实现自动同步

## 7. 安全考虑

- 密码使用 argon2 加密存储
- Token 使用 RS256 签名
- 敏感操作需要重新验证密码
- Token 黑名单机制支持主动撤销
- 离线 token 采用更严格的存储加密
