# 教育管理系统 - 系统重构设计方案

**版本**：v1.0
**日期**：2026-04-01
**状态**：待用户审核

## 一、系统目标

### 1.1 核心需求
- 小型教育管理系统（<100人，教职工为主）
- 全中文界面、中文数据库表名字段
- 低耦合、易维护、可扩展
- 完整的权限管理体系

### 1.2 设计原则
- **简单优先**：不引入过度复杂的架构
- **领域驱动**：按业务领域划分模块
- **接口清晰**：API 与实现解耦
- **文档完整**：所有变更均有中文文档

## 四、前端解耦方案

### 4.1 API 层重构

**目标**：将 API 调用从业务逻辑中分离

**目录结构**：
```
src/api/
├── index.ts                 # 统一导出
├── request.ts              # Axios 实例配置
├── 系统/
│   ├── 用户.ts              # 用户 CRUD
│   ├── 角色.ts              # 角色 CRUD
│   ├── 菜单.ts              # 菜单 CRUD
│   └── 字典.ts              # 字典 CRUD
└── 教育/
    ├── 教师.ts
    ├── 学生.ts
    ├── 班级.ts
    ├── 课程.ts
    └── 成绩.ts
```

**示例代码**：

```typescript
// src/api/系统/用户.ts
import { request } from '../request'

export interface 用户DTO {
  id?: number
  yonghuming: string
  mima?: string
  nicheng?: string
  shoujihao?: string
  youxiang?: string
  xingbie?: string
  zhuangtai?: number
}

export const 用户API = {
  /** 获取用户列表 */
  获取列表: (params?: Partial<用户DTO>) =>
    request.get<用户DTO[]>('/系统用户', { params }),

  /** 获取用户详情 */
  获取详情: (id: number) =>
    request.get<用户DTO>(`/系统用户/${id}`),

  /** 新增用户 */
  新增: (data: 用户DTO) =>
    request.post('/系统用户', data),

  /** 修改用户 */
  修改: (id: number, data: 用户DTO) =>
    request.put(`/系统用户/${id}`, data),

  /** 删除用户 */
  删除: (id: number) =>
    request.delete(`/系统用户/${id}`),

  /** 重置密码 */
  重置密码: (id: number) =>
    request.post(`/系统用户/${id}/重置密码`),
}
```

### 4.2 Composables 层重构

**目标**：将业务逻辑从 Vue 组件中抽取

**目录结构**：
```
src/composables/
├── use用户管理.ts
├── use角色权限.ts
├── use菜单配置.ts
├── use教师管理.ts
├── use学生管理.ts
├── use班级管理.ts
├── use课程管理.ts
└── use成绩管理.ts
```

**示例代码**：

```typescript
// src/composables/use用户管理.ts
import { ref, computed } from 'vue'
import { 用户API, 用户DTO } from '@/api/系统/用户'
import { 消息提示 } from '@/utils/消息'

export function use用户管理() {
  const 加载中 = ref(false)
  const 用户列表 = ref<用户DTO[]>([])
  const 当前用户 = ref<用户DTO | null>(null)

  const 用户数量 = computed(() => 用户列表.value.length)

  /** 加载用户列表 */
  async function 加载列表(params?: Partial<用户DTO>) {
    加载中.value = true
    try {
      用户列表.value = await 用户API.获取列表(params)
    } finally {
      加载中.value = false
    }
  }

  /** 保存用户（新增或修改） */
  async function 保存用户(data: 用户DTO) {
    加载中.value = true
    try {
      if (data.id) {
        await 用户API.修改(data.id, data)
        消息提示().success('修改成功')
      } else {
        await 用户API.新增(data)
        消息提示().success('新增成功')
      }
      await 加载列表()
    } finally {
      加载中.value = false
    }
  }

  /** 删除用户 */
  async function 删除用户(id: number) {
    await 用户API.删除(id)
    消息提示().success('删除成功')
    await 加载列表()
  }

  return {
    加载中,
    用户列表,
    当前用户,
    用户数量,
    加载列表,
    保存用户,
    删除用户,
  }
}
```

### 4.3 视图层简化

## 五、权限模块设计

### 5.1 权限模型

采用 **RBAC** 模型（Role-Based Access Control）：

```
用户 ──── 属于 ────> 角色 ──── 拥有 ────> 权限
                        │
                        └─── 关联 ────> 数据范围
```

### 5.2 权限类型

| 权限类型 | 说明 | 示例 |
|---------|------|------|
| 菜单权限 | 是否显示菜单 | `用户管理:查看` |
| 操作权限 | 按钮级别控制 | `用户管理:新增` |
| 数据权限 | 可见数据范围 | 本部门/本人/全部 |

### 5.3 权限控制实现

#### 后端（NestJS）

```typescript
// roles.decorator.ts
export const Roles = (...roles: string[]) => SetMetadata('roles', roles);

// roles.guard.ts
@Injectable()
export class RolesGuard implements CanActivate {
  canActivate(context: ExecutionContext): boolean {
    const roles = Reflect.getMetadata('roles', context.getHandler())
    if (!roles) return true

    const user = context.switchToHttp().getRequest().user
    return user.roles.some(role => roles.includes(role.jiaose_quanxian_fuzhufu))
  }
}

// 使用示例
@Get(':id')
@Roles('系统用户:查看')
async 获取用户(@Param('id') id: number) {
  return this.userService.findById(id)
}
```

#### 前端（指令）

```typescript
// v-permission.ts 指令
export const vPermission = {
  mounted(el: HTMLElement, binding: DirectiveBinding) {
    const { value } = binding
    const userPermissions = useUserStore().permissions

    if (value && !userPermissions.includes(value)) {
      el.parentNode?.removeChild(el)
    }
  }
}

// 使用示例
<template>
  <n-button v-permission="'系统用户:新增'" type="primary">
    新增用户
  </n-button>
</template>
```
