# 火柴人寻宝游戏 - 设计文档

## 项目概述

- **项目名称**: 火柴人寻宝 (Stickman Diamond Hunter)
- **类型**: HTML5 2D 像素风游戏
- **核心玩法**: 玩家控制火柴人在随机生成的地图中寻找钻石，躲避敌人，收集钥匙解锁高级钻石
- **目标用户**: 休闲游戏玩家

---

## 游戏机制

### 1. 控制方式

| 操作 | 输入 |
|------|------|
| 移动 | W/A/S/D 或方向键 |
| 确认 | 上/下/左/右按钮（点击或键盘） |

### 2. 地图系统

- **生成方式**: 随机生成，包含以下元素
  - 空地（可通行）
  - 障碍物：树木、石头（不可通行）
  - 边界（不可穿越）
- **关卡递进**:
  - 第1关: 15x15 格子，3个普通钻石，1个高级钻石，1把钥匙，1个敌人
  - 每关增加: 地图+2格，钻石+2，敌人+1
  - 最大关卡: 10关

### 3. 物品系统

| 物品 | 符号 | 获取方式 | 效果 |
|------|------|----------|------|
| 普通钻石 | 💎 | 触碰自动拾取 | +10分 |
| 高级钻石 | 💠 | 靠近后按确认键 | +30分，需要钥匙 |
| 钥匙 | 🔑 | 散落在地图 | 解锁高级钻石 |

### 4. 敌人系统

- **敌人类型**: 巡逻敌人（随机方向移动）
- **行为**:
  - 在地图随机位置生成
  - 每2秒随机改变方向
  - 碰到玩家扣1条命
- **生命值**: 3条命

### 5. 积分与通关

- **得分**:
  - 普通钻石: 10分
  - 高级钻石: 30分
  - 关卡加成: 分数 × (1 + 关卡数 × 0.1)
- **通关条件**: 收集所有钻石
- **失败条件**: 生命值归零
- **最高分**: 记录历史最高分

## 界面设计

### 1. 游戏界面布局

```
┌─────────────────────────────────────┐
│  关卡: 1  生命: ♥♥♥  分数: 0       │
├─────────────────────────────────────┤
│                                     │
│         [游戏地图区域]               │
│           15x15 格子                │
│                                     │
├─────────────────────────────────────┤
│  [上][下] [左][右]  确认按钮        │
│        控制按钮区域                  │
└─────────────────────────────────────┘
```

### 2. 视觉风格

- **火柴人**: 黑色线条绘制，头+身体+四肢
- **地图**: 绿色草地为底，灰色障碍物
- **UI**: 简约像素风格，黑色边框
- **动画**: 移动时有简单的动画效果

### 3. 游戏状态

- **开始界面**: 显示标题、开始按钮、最高分
- **游戏中**: 显示地图、分数、生命值
- **暂停**: 按ESC暂停游戏
- **通关**: 显示得分，进入下一关
- **结束**: 显示最终分数，重新开始按钮

---

## 技术实现

- **技术栈**: 纯 HTML5 + Canvas + JavaScript
- **单文件**: 所有代码在一个 HTML 文件中
- **无需依赖**: 不需要任何外部库
