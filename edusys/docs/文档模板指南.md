# 文档模板指南

本文档定义了 Fayzedu 项目的文档模板标准和编写规范。

## 模板分类

### 1. 指南类文档模板

**适用文档**: 快速开始、使用指南、维护指南等

```markdown

[简要说明文档的目的和主要内容]

- [功能说明]
- [使用步骤]
- [注意事项]

### [子功能2]
- [功能说明]
- [使用步骤]
- [注意事项]

## [主要功能/模块2]

### [子功能1]
- [功能说明]
- [使用步骤]
- [注意事项]

**Q: [问题1]**

- [相关文档1](链接)
- [相关文档2](链接)

*最后更新: [日期]*
```

## 编写规范

### 文件命名
- 使用中文命名，便于查找
- 格式: `[主题]-[类型].md`
- 示例: `API文档.md`, `架构设计文档.md`, `快速开始指南.md`

### 内容结构
- **标题层级**: 使用规范的 Markdown 标题层级
- **代码块**: 使用适当的语言标签
- **表格**: 使用 Markdown 表格语法
- **链接**: 使用相对路径链接

### 格式要求
- **语言**: 使用简体中文
- **编码**: UTF-8
- **换行**: 使用统一的换行符
- **缩进**: 使用统一的缩进方式

### 更新维护
- **版本管理**: 使用 Git 进行版本管理
- **更新记录**: 在文档末尾添加更新记录
- **链接检查**: 定期检查文档链接的有效性

### 目录结构规范

```

│   ├── 需求采集.md
```

### 文档评审流程

1. **初稿编写**: 作者完成文档初稿
2. **内部评审**: 邀请相关人员评审（技术评审/业务评审）
3. **修订完善**: 根据评审意见修订文档
4. **最终确认**: 相关负责人确认通过
5. **版本发布**: 合并到主分支，标记版本

### 变更记录格式

每次文档更新时，在文件末尾添加：

```markdown
## 变更记录

| 版本 | 日期 | 作者 | 变更内容 |
|------|------|------|----------|
| v1.1 | 2026-04-03 | 张三 | 新增API接口文档 |
| v1.0 | 2026-03-27 | 李四 | 初始版本 |
```

## 最佳实践

### 1. 内容组织
- 按逻辑顺序组织内容
- 使用清晰的标题层级
- 提供足够的示例和说明

### 2. 可读性
- 使用简洁明了的语言
- 避免技术术语的过度使用
- 提供必要的上下文说明

### 3. 维护性
- 保持文档的时效性
- 及时更新变更的内容
- 建立文档审查机制

- **Markdown**: 主要文档格式
