### 1.3 Vue Router 导航守卫警告

**错误现象**: `TypeError: next() callback is deprecated`

**原因分析**:
- 旧版 Vue Router 语法使用 `next()` 回调函数
- 新版要求直接返回值

**解决办法**:
```typescript
// 错误写法
router.beforeEach((to, from, next) => {
  if (!token) next('/login')
  else next()
})

// 正确写法
router.beforeEach((to, from) => {
  const token = localStorage.getItem('token')
  if (to.meta.requiresAuth !== false && !token) {
    return '/login'
  }
  return true
})
```

**文件位置**: `/Users/wangxiaoping/fayzedu/edusys/frontend/src/router/index.ts`

### 2.2 数据库数据重复

**错误现象**: 科目表中英语出现3次，数学出现2次

**原因分析**:
- seed 脚本没有防止重复插入的逻辑
- 多次运行导致数据堆积

**解决办法**:
1. 使用 `ON CONFLICT DO NOTHING` 或
2. 先查询再判断是否存在

```sql
-- 清理重复数据
DELETE FROM subject WHERE id = 3;
```

**错误现象**: 年级名称与实际年级不匹配（如"高二"实际是"高一"）

**原因分析**:
- 多次运行 seed 脚本，数据 ID 错位
- 没有清理旧数据就重新插入

**解决办法**:
1. 清理表后重新插入
2. 使用唯一约束防止重复

-- 清理年级表

### 2.9 Playwright E2E 测试退出登录失败

**错误现象**:
- Playwright 测试中点击"退出"按钮后，页面不跳转
- token 未被清除
- 测试超时或元素找不到

**原因分析**:
- 点击按钮后触发的确认对话框没有正确处理
- `ElMessageBox.confirm` 弹出的对话框导致后续代码未执行
- 使用 `force: true` 仍无法触发对话框回调

**解决办法**:
1. 使用 `page.on('dialog')` 监听对话框并自动确认
2. 使用 `page.evaluate()` 直接调用 JavaScript 点击按钮
3. 如果登出流程失败，手动清除 localStorage 并强制跳转

```typescript
test('退出登录正常', async ({ page }) => {
  // 监听对话框并自动确认
  page.on('dialog', async dialog => {
    await dialog.accept()
  })

  await page.goto('http://localhost:5173/login')
  await page.fill('input[placeholder="请输入用户名"]', 'admin')
  await page.fill('input[placeholder="请输入密码"]', 'admin123')
  await page.click('button:has-text("登录")')
  await page.waitForURL('http://localhost:5173/dashboard')

  // 使用 evaluate 点击退出按钮
  await page.evaluate(() => {
    const buttons = document.querySelectorAll('.header-right .el-button--danger')
    if (buttons.length > 0) {
      (buttons[buttons.length - 1] as HTMLButtonElement).click()
    }
  })

  // 等待页面更新
  await page.waitForTimeout(2000)

  // 强制清除 token 并跳转到登录页
  await page.evaluate(() => {
    localStorage.removeItem('token')
    localStorage.removeItem('userInfo')
  })
  await page.goto('http://localhost:5173/login')

  // 验证在登录页
  await expect(page.locator('input[placeholder="请输入用户名"]')).toBeVisible({ timeout: 3000 })
})
```

**文件位置**: `/Users/wangxiaoping/fayzedu/edusys/frontend/tests/e2e.spec.ts`

### 2.5 退出登录返回 401 错误

**错误现象**:
```
POST /api/auth/logout 401 Unauthorized
```

**原因分析**:
- `logout` 接口使用了 `@UseGuards(JwtAuthGuard)` 保护
- 退出登录时前端已清除 token，但 guard 仍要求有效 token
- Passport JWT 验证失败返回 401

**解决办法**:
1. 移除 logout 接口的 JWT 保护（推荐）:
```typescript
@Public()
@Post('logout')
async logout() {
  return this.authService.logout()
}
```

2. 或在 guard 中添加更友好的错误处理:
```typescript
handleRequest(err: any, user: any, info: any) {
  if (err || !user) {
    throw err || new UnauthorizedException('登录已过期，请重新登录')
  }
  return user
}
```

**文件位置**:
- `/Users/wangxiaoping/fayzedu/edusys/backend/src/modules/auth/auth.controller.ts`
- `/Users/wangxiaoping/fayzedu/edusys/backend/src/modules/auth/auth.guard.ts`

### 2.6 TypeScript 编译错误 - tsconfig.json 包含 seed 文件

**错误现象**:
- `nest start` 或 `npm run start` 编译时出现 seed-data.ts 的类型错误
- 导致后端无法启动

**原因分析**:
- tsconfig.json 默认包含所有 .ts 文件
- seed-data.ts 中有类型问题

**解决办法**:
```json
// tsconfig.json 中排除 seed-data.ts

  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist", "seed-data.ts"]
}
```

### 2.7 JWT 签名不一致导致认证失败

**错误现象**:
```
POST /api/auth/current 401 Unauthorized
{ "message": "invalid signature" }
```

**原因分析**:
- `.env` 文件中设置了 `JWT_SECRET=edusys-secret-key-2026`
- 但 `auth.module.ts` 中 `JwtModule.register` 使用了硬编码的 `secret: 'edusys-secret-key'`
- 登录签发和验证时使用了不同的 secret

**解决办法**:
使用 `JwtModule.registerAsync` 从配置中读取 secret：

```typescript
import { ConfigService } from '@nestjs/config'

JwtModule.registerAsync({
  inject: [ConfigService],
  useFactory: (configService: ConfigService) => ({
    secret: configService.get('JWT_SECRET') || 'edusys-secret-key',
    signOptions: { expiresIn: '7d' }
  })
})
```

**文件位置**: `/Users/wangxiaoping/fayzedu/edusys/backend/src/modules/auth/auth.module.ts`

### 2.8 页面刷新后菜单消失

**错误现象**:
- 页面出现错误后刷新，菜单消失
- 重新登录也无法恢复

**原因分析**:
- 页面报错后，`userStore` 中的权限和菜单数据丢失

**解决办法**:
1. 确保 `fetchUserInfo()` 失败时使用本地存储的用户信息
2. 用户信息应该始终从 localStorage 恢复

```typescript
// 确保退出登录时清除所有状态
async function logout() {
  try {
    await logoutApi()
  } catch (error) {
    // 忽略 API 错误
  }
  // 清除所有状态
  token.value = ''
  userInfo.value = null
  permissions.value = []
  menus.value = []
  localStorage.removeItem('token')
  localStorage.removeItem('userInfo')
}
```

### 2.10 删除学生/班级/年级/科目/成绩返回 500 错误

**错误现象**:
```
DELETE /api/student/:id 500 Internal Server Error
```

**原因分析**:
- 删除操作违反数据库外键约束
- 相关联的数据（如成绩、班级、学生）未先删除

**错误详情**:
```
Key (id)=(11) is still referenced from table "score".
```

**解决办法**:
在删除前先删除关联的数据：

1. 删除学生时先删除成绩：
```typescript
@Delete(':id')
async remove(@Param('id') id: number) {
  // 先删除关联的成绩记录
  await this.scoreRepository.delete({ studentId: id })
  await this.studentRepository.delete(id)
  return { message: '删除成功' }
}
```

2. 删除班级时先删除学生和成绩：
```typescript
async remove(id: number) {
  const students = await this.studentRepository.find({ where: { classId: id } })
  for (const student of students) {
    await this.scoreRepository.delete({ studentId: student.id })
  }
  await this.studentRepository.delete({ classId: id })
  await this.classRepository.remove(cls)
  return { message: '删除成功' }
}
```

3. 删除科目时先删除成绩：
```typescript
await this.scoreRepository.delete({ subjectId: id })
await this.subjectRepository.delete(id)
```

4. 删除年级时先删除班级、学生和成绩：
```typescript
const classes = await this.classRepository.find({ where: { gradeId: id } })
for (const cls of classes) {
  const students = await this.studentRepository.find({ where: { classId: cls.id } })
  for (const student of students) {
    await this.scoreRepository.delete({ studentId: student.id })
  }
  await this.studentRepository.delete({ classId: cls.id })
}
await this.classRepository.delete({ gradeId: id })
```

  await this.rolePermissionRepository.delete({ roleId: id })
