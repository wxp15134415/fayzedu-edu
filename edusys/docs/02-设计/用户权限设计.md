# 用户与权限管理模块设计

## 概述

本文档定义了学校管理系统中「用户与权限管理」模块的设计方案，包括用户角色体系、权限模型、技术实现等内容。

## 需求背景

学校管理系统需要支持多种用户角色的后台管理，超级管理员可以管理整个系统，管理员可以管理特定年级/班级的数据，教师和家长则根据角色分配相应的访问权限。

## 设计目标

- 支持 5 种用户角色：超级管理员、管理员、教师、学生、家长
- 实现「角色 + 权限组」的权限管理模式
- 支持按年级/班级进行数据范围控制
- 实现基本的账号密码登录功能
- 实现页面级访问控制

## 用户角色体系

### 角色定义

| 角色 | 说明 | 数量 | 权限级别 |
|------|------|------|----------|
| 超级管理员 | 系统最高权限，负责系统配置 | 1 | 全局 |
| 管理员 | 学校日常运营管理 | 多个 | 年级/班级 |
| 教师 | 任课教师，可查看/管理所教班级学生 | 多个 | 班级 |
| 学生 | 可登录查看自己的成绩、考勤等 | 多个 | 个人 |
| 家长 | 关联学生，可查看孩子情况 | 多个 | 关联学生 |

### 角色说明

1. **超级管理员**：系统唯一账号，拥有所有权限，可以管理所有模块
2. **管理员**：可分配管辖的年级/班级，可以对管辖范围内的数据进行管理
3. **教师**：可查看所教授班级的学生信息、成绩、考勤等
4. **学生**：只能查看自己的信息，不能管理其他数据
5. **家长**：只能查看关联学生的信息，不能管理其他数据

## 权限模型

### 权限组设计

权限组是可自定义的权限集合，一个角色可以关联多个权限组。

预定义的权限组：

| 权限组 | 包含的页面权限 |
|--------|----------------|
| 用户管理 | 用户列表、用户新增、用户编辑、用户删除 |
| 角色管理 | 角色列表、角色新增、角色编辑 |
| 权限组管理 | 权限组列表、权限组新增、权限组编辑 |
| 学生管理 | 学生列表、学生详情 |
| 成绩管理 | 成绩查看、成绩录入 |
| 考勤管理 | 考勤查看、考勤录入 |
| 班级管理 | 班级列表、班级编辑 |

### 页面权限定义

每个页面对应一个唯一的权限标识，用户必须拥有该权限标识才能访问对应的页面。

| 权限标识 | 对应页面 |
|----------|----------|
| user:list | 用户列表 |
| user:add | 用户新增 |
| user:edit | 用户编辑 |
| user:delete | 用户删除 |
| role:list | 角色列表 |
| role:add | 角色新增 |
| role:edit | 角色编辑 |
| permission:list | 权限组列表 |
| permission:add | 权限组新增 |
| permission:edit | 权限组编辑 |

### 数据范围控制

用户只能访问其管辖范围内的数据：

- **超级管理员**：可访问所有数据
- **管理员**：可访问分配给的年级/班级数据
- **教师**：可访问所教授班级的数据
- **学生**：只能访问自己的数据
- **家长**：只能访问关联学生的数据

### 前端页面设计

#### 布局结构

- **整体布局**：混合式布局（左侧菜单 + 顶部导航）
- **侧边栏**：可折叠，包含主要功能菜单
- **顶部**：Logo、面包屑、用户信息、登出按钮
- **内容区**：页面主要内容

#### 核心页面

1. **登录页** - 账号密码登录
2. **用户管理页** - 用户列表、新增、编辑、禁用
3. **角色管理页** - 角色列表、新增、编辑、分配权限
4. **权限组管理页** - 权限组列表、新增、编辑

### 关键流程

#### 登录流程

1. 用户输入用户名和密码
2. 前端发送登录请求到后端
3. 后端验证用户名和密码
4. 验证通过后，生成 JWT token 返回给前端
5. 前端将 token 存储到本地，并跳转到首页

#### 权限验证流程

1. 用户登录后，前端获取用户信息和权限列表
2. 根据权限列表动态生成菜单
3. 用户访问页面时，检查是否有对应权限
4. 无权限则显示403页面

## 风险评估

1. **密码安全**：初始密码要求较低（≥6位），建议后续加强或由管理员强制要求修改
2. **数据隔离**：需要确保数据范围控制在数据库层实现，防止越权访问
3. **Token安全**：使用 JWT，需要配置合理的过期时间
