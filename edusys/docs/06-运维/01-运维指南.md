## 1. 监控与告警

### 1.1 系统监控

| 监控项 | 指标 | 告警阈值 |
|--------|------|----------|
| CPU 使用率 | cpu_percent | > 80% |
| 内存使用率 | memory_percent | > 85% |
| 磁盘使用率 | disk_percent | > 90% |
| 后端服务 | http://localhost:3000 | 不可用 |
| 数据库连接 | PostgreSQL | 失败 |

### 1.2 日志管理

```bash
# PM2 日志
pm2 logs edusys-backend --lines 100

# Nginx 访问日志
tail -f /var/log/nginx/access.log

# Nginx 错误日志
tail -f /var/log/nginx/error.log
```

## 2. 日常维护

### 2.1 定期任务

| 任务 | 频率 | 说明 |
|------|------|------|
| 日志清理 | 每天 | 清理超过7天的日志 |
| 数据库备份 | 每天 | 每日自动备份 |
| 安全更新 | 每周 | 检查并更新系统包 |
| 性能检查 | 每周 | 检查系统性能 |

### 2.2 日志清理脚本

```bash
#!/bin/bash
# clean_logs.sh

# 清理 PM2 日志（超过7天）
pm2 flush

# 清理 Nginx 日志
find /var/log/nginx -name "*.log" -mtime +7 -delete
```

### 2.3 自动备份

```bash
# 添加 crontab
crontab -e

# 每天凌晨2点备份数据库
0 2 * * * /home/deploy/backup.sh
```

## 3. 故障处理

### 3.1 服务故障

```bash
# 重启后端服务
pm2 restart edusys-backend

# 重启 Nginx
sudo systemctl restart nginx

# 重启数据库
sudo systemctl restart postgresql
```

### 3.2 常见故障

| 故障现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 页面无法访问 | Nginx 未运行 | 检查 Nginx 状态并重启 |
| API 请求失败 | 后端服务未运行 | 检查并重启后端服务 |
| 登录失败 | 数据库连接问题 | 检查数据库配置和连接 |
| 502 Bad Gateway | 后端服务异常 | 查看后端日志排查问题 |

## 4. 安全建议

### 4.1 系统安全

- 定期更新系统和依赖包
- 配置防火墙规则
- 使用 HTTPS 加密传输
- 定期更换 JWT Secret

### 4.2 数据安全

- 定期备份数据库
- 限制数据库访问权限
- 启用数据库审计日志

## 5. 性能优化

### 5.1 后端优化

- 使用 PM2 集群模式
- 开启 Redis 缓存（可选）
- 配置适当的连接池大小

### 5.2 前端优化

- 启用 Gzip 压缩
- 配置静态资源缓存
- 使用 CDN 加速
