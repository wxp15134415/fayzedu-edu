# FayzEdu 教育管理系统 - 开发指南 v1.0

> 更新时间：2026-04-03

## 1. 开发环境

### 1.1 环境要求

| 软件 | 版本要求 |
|------|----------|
| Node.js | >= 18.x |
| pnpm | >= 8.x |
| PostgreSQL | >= 14.x |
| Git | >= 2.x |

### 1.2 环境变量配置

#### 后端 (.env)

```bash
# 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=your_password
DB_DATABASE=edusys

# JWT 配置
JWT_SECRET=your_jwt_secret_key
JWT_EXPIRES_IN=24h

# 服务配置
PORT=3000
```

#### 前端 (.env)

```bash
VITE_API_BASE_URL=http://localhost:3000
```

## 2. 安装与运行

### 2.1 后端安装

```bash
cd edusys/backend
pnpm install
```

### 2.2 后端运行

```bash
# 开发模式（热重载）
pnpm run start:dev

# 生产模式
pnpm run build
pnpm run start:prod
```

### 2.3 前端安装

```bash
cd edusys/frontend
pnpm install
```

### 2.4 前端运行

```bash
# 开发模式
pnpm dev

# 生产构建
pnpm build
```

## 3. 数据库初始化

### 3.1 创建数据库

```sql
CREATE DATABASE edusys;
```

### 3.2 运行迁移

TypeORM 会根据实体自动创建表，确保数据库配置正确后启动服务即可。

### 3.3 初始数据

系统启动后需要手动创建初始角色和权限：

```sql
-- 插入初始角色
INSERT INTO sys_role (role_name, role_code, role_desc, is_system)
VALUES ('超级管理员', 'super', '系统最高权限', 1);

-- 插入初始权限
INSERT INTO sys_permission (permission_name, permission_code, permission_desc)
VALUES ('用户管理', 'user:manage', '用户管理权限'),
       ('角色管理', 'role:manage', '角色管理权限'),
       ('权限管理', 'permission:manage', '权限管理权限');

-- 插入初始管理员用户 (密码: 123456)
-- 注意：实际密码需要 bcrypt 加密
```

## 4. 模块开发指南

### 4.1 创建新模块

使用 NestJS CLI：

```bash
nest g module modules/xxx
nest g controller modules/xxx
nest g service modules/xxx
```

### 4.2 创建实体

```typescript
import { Entity, PrimaryGeneratedColumn, Column } from 'typeorm';

@Entity('table_name')
export class EntityName {
  @PrimaryGeneratedColumn('increment', { type: 'bigint' })
  id: number;

  @Column({ type: 'varchar', length: 50 })
  fieldName: string;
}
```

### 4.3 创建 DTO

```typescript
import { IsString, IsNotEmpty } from 'class-validator';

export class CreateXxxDto {
  @IsNotEmpty()
  @IsString()
  fieldName: string;
}
```

## 8. 常见问题

### 8.1 数据库连接失败

检查 .env 配置是否正确，确保 PostgreSQL 服务已启动。

### 8.2 依赖安装失败

建议使用 pnpm，确保 Node.js 版本符合要求。

### 8.3 端口被占用

修改 .env 中的 PORT 配置，或关闭占用端口的进程。
