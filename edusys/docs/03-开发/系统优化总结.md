## 技术决策记录

### Redis 缓存 - 暂不实现 (2026-04-04)

**决策**: 暂时不做 Redis 缓存

**原因**:
- 当前数据量不大，不需要缓存
- 增加系统复杂度（需处理穿透/击穿/雪崩）
- 维护成本高，多一个依赖需要运维

**未来引入条件**:
- 学生数据超过 10,000 条
- 成绩数据超过 100,000 条
- 出现明显的查询性能瓶颈

**预留方案**:
- 菜单树/年级列表: Cache-Aside + 30分钟 TTL
- 学生/成绩: TTL 过期（5-10分钟）
- 系统配置: Write-Through（变更立即更新）
